## 実装時のテストケース

P0完了時に必ず以下を確認：

```typescript
// tests/integration/2024-12.test.ts
import { parsePdf } from '@/lib/pdf/parser';
import { aggregate } from '@/lib/pdf/classifier';
import expected from '../fixtures/2024-12-expected.json';

describe('2024年12月分（2025年1月PDF）', () => {
  let result: AggregatedProperty[];

  beforeAll(async () => {
    const buffer = readFileSync('./tests/fixtures/2025年1月支払通知書.pdf');
    const parsed = await parsePdf(buffer.buffer);
    result = aggregate(parsed.rows);
  });

  it('邸数が18邸', () => {
    expect(result).toHaveLength(18);
  });

  it.each(expected.properties)('$propertyName の集計値が一致', (exp) => {
    const actual = result.find(p => p.propertyName === exp.propertyName);
    expect(actual).toBeDefined();
    expect(actual!.amountSales).toBe(exp.amountSales);
    expect(actual!.amountShaho).toBe(exp.amountShaho);
    expect(actual!.amountSeisanka).toBe(exp.amountSeisanka);
    expect(actual!.amountMaterial).toBe(exp.amountMaterial);
  });

  it('合計が期待通り', () => {
    const totalSales = result.reduce((s, p) => s + p.amountSales, 0);
    expect(totalSales).toBe(14_726_606);
  });
});
```

## 異常系のサンプル

以下のケースは現状サンプルが手元にない。実装後に発生したら、追加テストケースとして登録：

- 邸数が0または異常に多い（>50）PDF
- 工種名や邸名にUnicode変な文字が入っているケース
- スキャンPDF（画像）→ エラーで弾けるか
- 1ページしかない極小PDF
- 50ページ超の巨大PDF
